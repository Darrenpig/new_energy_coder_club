# ESP32-S3 无线烧录配置说明

## 概述

本文档记录了使用无线 DAP-Link 烧录器对 ESP32-S3 进行串口烧录的完整配置方案。

## 硬件连接

### 无线烧录器连接方式

- **电脑端**：无线 DAP-Link 发射器 → USB 连接到电脑
- **开发板端**：无线 DAP-Link 接收器 → 通过引脚连接到 ESP32-S3 的 UART

### 引脚连接

无线烧录器接收端需要连接到 ESP32-S3 的 UART0 引脚：

| 烧录器引脚 | ESP32-S3 引脚 | 说明 |
|-----------|--------------|------|
| TX        | GPIO44 (RX)  | 发送 → 接收 |
| RX        | GPIO43 (TX)  | 接收 → 发送 |
| GND       | GND          | 共地 |

**注意**：不是连接 USB 口，而是直接连接 UART 引脚！

## PlatformIO 配置

### platformio.ini 关键配置

```ini
[env:temple_project]

platform = espressif32
framework = arduino
board = esp32-s3-devkitc-1

;指定上传协议（烧录方式）
upload_protocol = esptool ;通过UART串口烧录

;无线烧录时禁用自动复位（需要手动进入下载模式）
upload_flags =
    --before=no_reset      ;禁用烧录前自动复位
    --after=hard_reset     ;烧录后硬件复位
    --connect-attempts=10  ;增加连接重试次数（适应无线延迟）
    --baud=115200         ;设置波特率

;串口监视器波特率
monitor_speed = 115200
```

### 配置说明

| 参数 | 作用 | 原因 |
|------|------|------|
| `upload_protocol = esptool` | 使用串口烧录协议 | ESP32 标准烧录方式 |
| `--before=no_reset` | 禁用自动复位 | 无线烧录器缺少 DTR/RTS 控制信号 |
| `--after=hard_reset` | 烧录后硬件复位 | 确保固件正常启动 |
| `--connect-attempts=10` | 增加连接重试次数 | 补偿无线传输延迟 |
| `--baud=115200` | 设置波特率 | 平衡速度和稳定性 |

## 烧录操作流程

### 1. 手动进入下载模式

由于无线烧录器没有 DTR/RTS 控制信号，需要手动操作：

1. **按住 BOOT 按钮**（GPIO0）
2. **按一下 EN 按钮**（复位）
3. **继续按住 BOOT 按钮**

### 2. 执行烧录命令

保持按住 BOOT 按钮，运行：

```bash
pio run -e temple_project -t upload --upload-port COM9
```

或在 VSCode 中按 `Ctrl+Alt+U`（需要先在底部状态栏选择正确的 COM 口）

### 3. 松开 BOOT 按钮

当看到终端显示 `Connecting...` 后，松开 BOOT 按钮。

### 4. 等待烧录完成

烧录成功后会显示：
```
Writing at 0x00010000... (100 %)
Wrote XXXXX bytes at 0x00010000 in X.X seconds
```

## 代码配置（串口通信）

### 使用 Serial0 而非 Serial

由于启用了 `ARDUINO_USB_CDC_ON_BOOT=1`，`Serial` 默认指向 USB CDC，需要使用 `Serial0` 访问 UART0：

```cpp
void setup() {
    // 使用 UART0 (GPIO43/44)，与无线烧录器通信
    Serial0.begin(115200, SERIAL_8N1, 44, 43); // RX=44, TX=43
    delay(500);
    
    Serial0.println("系统启动");
}

void loop() {
    // 读取串口输入
    while (Serial0.available()) {
        char c = Serial0.read();
        Serial0.print(c);
    }
}
```

### ESP32-S3 的两个串口

| 串口 | 物理接口 | 代码中的名称 | 用途 |
|------|---------|-------------|------|
| USB CDC | GPIO19/20 (内置USB) | `Serial` | USB 虚拟串口 |
| UART0 | GPIO43/44 | `Serial0` | 硬件串口（无线烧录器连接） |

## 常见问题

### 1. 烧录失败：`No serial data received`

**原因**：ESP32 未进入下载模式

**解决**：
- 确保按住 BOOT 按钮直到看到 "Connecting..."
- 增加 `--connect-attempts` 参数值

### 2. 烧录失败：`Could not open COM9`

**原因**：串口被占用或不存在

**解决**：
- 关闭所有串口监视器
- 在设备管理器中确认正确的 COM 口号
- 重新插拔无线烧录器

### 3. 串口监视器无输出

**原因**：代码使用了错误的串口

**解决**：
- 确保代码中使用 `Serial0` 而非 `Serial`
- 确认串口监视器连接到正确的 COM 口
- 检查波特率是否为 115200

### 4. 无线烧录速度慢

**原因**：无线传输延迟

**解决**：
- 降低波特率：`--baud=115200` 或更低
- 减少无线干扰源
- 必要时切换到有线模式

## 有线 vs 无线烧录对比

| 特性 | 有线烧录 | 无线烧录 |
|------|---------|---------|
| 连接方式 | USB 直连 | 无线 DAP-Link |
| 进入下载模式 | 自动（DTR/RTS） | 手动（按 BOOT+EN） |
| 烧录速度 | 快 | 较慢 |
| 稳定性 | 高 | 受无线环境影响 |
| 便利性 | 需要插线 | 无线自由 |

## 多环境配置（可选）

如果需要同时支持有线和无线烧录，可以创建两个环境：

```ini
;有线烧录环境（默认）
[env:temple_project]
upload_protocol = esptool
;自动复位，无需手动操作

;无线烧录环境
[env:temple_project_wireless]
upload_protocol = esptool
upload_flags =
    --before=no_reset
    --after=hard_reset
    --connect-attempts=10
    --baud=115200
```

切换环境：
- 有线：`pio run -e temple_project -t upload`
- 无线：`pio run -e temple_project_wireless -t upload --upload-port COM9`

## 总结

### 核心要点

1. **硬件连接**：无线烧录器接收端连接 UART 引脚（TX/RX/GND），不是 USB 口
2. **配置关键**：`upload_protocol = esptool` + `--before=no_reset`
3. **操作关键**：手动进入下载模式（按住 BOOT + 按 EN）
4. **代码关键**：使用 `Serial0` 访问 UART0

### 为什么无线烧录需要特殊配置？

- ESP32 串口烧录需要控制 EN（复位）和 GPIO0（启动模式）引脚
- 标准 USB 转串口芯片通过 DTR/RTS 信号自动控制这些引脚
- 无线 DAP-Link 的虚拟串口**不传输 DTR/RTS 信号**
- 因此需要禁用自动复位（`--before=no_reset`）并手动操作
